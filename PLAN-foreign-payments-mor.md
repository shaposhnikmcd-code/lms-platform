# План: закордонні платежі через Merchant of Record (Paddle)

> Мета: іноземці (США, Німеччина, інші) можуть платити за курси/пакети/Річну, бо у WayForPay масові відмови по закордонних картках. Українці лишаються на WayForPay без змін.

## Рішення (зафіксовано)

- **Основна система:** Paddle. Запасна, якщо Paddle відмовить: Dodo Payments.
- **Lemon Squeezy виключено** — платить тільки на bank/PayPal, Payoneer немає, PayPal UA не приймає комерцію, + мігрує в Stripe.
- **Ланцюг грошей:** іноземець → Paddle → Payoneer → український ФОП (3 група, 5% ЄП + 1% військовий збір).
- **Фаза 1 для закордону — тільки разові покупки** (курси, пакети, Річна як разовий платіж на рік). Рекурентна місячна для іноземців — окремо у фазі 5 (складніше, не зараз).
- **Валюта для іноземців:** USD (EUR — опційно пізніше).
- **Польський JDG/Stripe не чіпаємо** — виручка інституту туди не йде.
- Принцип у формі замовлення: `країна === UA → WayForPay`, інакше → Paddle.

## Чому це безпечно архітектурно

Сайт уже все вміє: WayForPay працює через підписаний webhook, а видача доступу винесена в універсальну функцію `provisionPayment()` (lib/paymentProvisioning.ts), яка не залежить від платіжки. Paddle перевикористає рівно її. Велике переписування не потрібне — це другий канал поруч з WayForPay.

## Фаза 1 — Реєстрація Paddle (без коду, робить власник)

Це як колись підключення WayForPay: зареєструвати мерчанта, не «бюрократія заради бюрократії».

1. Створити акаунт Paddle, вказати дані українського ФОПа.
2. Підтвердити домен сайту (domain verification).
3. Підключити Payoneer як спосіб виплат (рахунок Payoneer на ФОП). Мінімум виплати $100, раз на місяць до 15-го.
4. У заявці наголос: **UIMP — власна освітня платформа з авторськими курсами психотерапії + річне навчання, не реселл чужого контенту.** Це ключовий критерій Paddle (вони відхиляють реселл і «не свій» контент).
5. Створити в Paddle Products під кожен курс/пакет/Річну з цінами **в USD**.

➡️ **Якщо Paddle відмовляє** (через «курси» чи відсутність процесинг-історії) — перемикаємось на Dodo Payments, решта плану та сама.

**Вихід фази:** схвалений акаунт + Payoneer прив'язаний + продукти з USD-цінами заведені.

## Фаза 2 — Код інтеграції (окрема сесія)

### ✅ Зроблено (серверна частина, 2026-06-05) — компілюється чисто (tsc 0 помилок)

1. **БД:** поле `paymentProvider` (`@default("wayforpay")`) додано в `Payment`, db:push на dev-branch застосовано. `externalRef` (Paddle txn id) уже існував.
2. **`lib/paddle.ts`** — креди з env, `verifyPaddleSignature` (HMAC-SHA256, заголовок `Paddle-Signature`), `createPaddleTransaction`.
3. **`lib/paddleProducts.ts`** — мапа productKey → Paddle Price ID з env `PADDLE_PRICE_MAP`.
4. **`app/api/mor/route.ts`** — checkout: price lookup → user upsert → dup-guard → bundle CHOICE_FREE валідація → `Payment(PENDING, provider=paddle, currency=USD)` → `createPaddleTransaction` → повертає `checkoutUrl`. Поки env порожні — віддає 503 `paddle_not_configured` (очікувано).
5. **`app/api/mor/webhook/route.ts`** — підпис SHA256 → claim-then-act flip `PAID` (+ USD-сума з `grand_total`) → **та сама `provisionPayment()`** + bundle-email + dedupe-лог `source='paddle'`.

### ⏳ Лишилось у Фазі 2 (наступна сесія, потребує акаунта Paddle)

6. **Env (після Фази 1):** `PADDLE_API_KEY`, `PADDLE_WEBHOOK_SECRET`, `PADDLE_ENV` (sandbox/production), `PADDLE_PRICE_MAP` (JSON: productKey→pri_...). Ключі: курс `course:<slug>`, пакет `bundle_<slug>`, річна `yearly-program`.
7. **Клієнт — роутинг за країною** у CoursePurchaseModal: UA → WayForPay (як зараз), інакше → POST `/api/mor` → редірект на `checkoutUrl`.
8. **Webhook URL** зареєструвати в Paddle: `https://uimp.com.ua/api/mor/webhook` (+ pre для тесту).

### Скоуп і дельта

- **Скоуп Фази 1: КУРСИ + ПАКЕТИ.** Річна через Paddle — Фаза 2b (підписочна логіка окремо; зараз роут її відхиляє).
- Промокоди для закордону поки НЕ застосовуються (знижки — через Paddle discount codes пізніше).
- Підпис SHA256 (не MD5), валюта USD, сума зберігається в центах із Paddle `grand_total`.

## Податкова пам'ятка (ФОП 3 група)

- Навантаження **6%** = 5% єдиний + 1% військовий збір. Плюс фіксований ЄСВ ~1 760 грн/міс.
- Дохід визнається в день надходження на Payoneer (курс НБУ), переказати на гривневий рахунок ФОПа до кінця кварталу.
- Paddle = merchant of record → для тебе це експорт послуг агрегатору, ПДВ ЄС / податки США Paddle закриває сам. Перевірити, що КВЕД покриває освітні/інформаційні послуги.

## Відкриті ризики

1. **Онбординг Paddle** — головний ризик, на вході. Не загроза грошам, просто можуть не взяти → план Б (Dodo).
2. **Заморозка виплати** при різких стрибках/чарджбеках/скаргах — рідко при чесній торгівлі своїм контентом.
2. **Чарджбеки** — як і у WayForPay, частина будь-якого еквайрингу.
